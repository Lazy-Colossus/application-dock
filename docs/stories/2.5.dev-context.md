# Story 2.5 — Dev Context

## Pre-flight findings

### What already exists (reuse)
- `session_repo.write_session(path, session)` — path-agnostic atomic write. Ready for finalise.
- `session_repo.read_in_progress()` / `delete_in_progress()` — both in place.
- `archery_service.generate_session_label()` — NOT modified; a private `_pick_finalise_label` helper added instead (simpler than extending the public function).
- `useArcherySessionStore` — has `session`, `loading`, `error`, `saveTarget`. Needs `finaliseSession`.
- `ScoringBoardPage.vue` — already has `confirmedCount`; just add the banner block.
- `routes.ts` — add `/archery/results` as child of MainLayout, same as setup/scoring routes.

### Key decisions
1. `_pick_finalise_label(base, prefer)` — private helper in `archery_service.py`. Prefers the original label if free; otherwise finds next suffix from finalised files only (ignores in-progress file since we're replacing it).
2. `finalise_in_progress()` raises `FileNotFoundError` → router maps to 404.
3. `finalise_in_progress()` uses `model_copy(update=...)` for immutable update.
4. `useScores.ts` composable for `totalFor` + `ranked` — keeps store lean, reusable in Story 4.2.
5. `finaliseSession` in store: sets `error`, re-throws so the page can navigate or not.
6. Confirm dialog in `ResultsPage` is a local `finaliseDialogOpen` ref; uses `q-dialog`.
7. Results table: 18 body rows (numbered 1–18), `—` for missing targets. Summary ranking is in `ResultsPage`, not `ResultsTable`.
8. Tie-break: `localeCompare(…, { sensitivity: 'base' })` for case-insensitive alphabetical (mirrors CC-3).

### Risk register
- `model_copy(update=...)` — Pydantic v2 API; verify it doesn't clone `targets` shallowly. `model_copy` does a shallow copy by default; `targets` is a list of Pydantic models — fine for our read-then-write pattern.
- `q-dialog` stub in tests renders slot always; gate dialog content with `v-if="finaliseDialogOpen"` OR test the stub's v-model prop. Use the same approach as ScoreEntryPanel spec: override `q-dialog` stub locally with `v-model` passthrough.
