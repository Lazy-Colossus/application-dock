# Story 7.5: Bottom-Anchored Board Controls & Persistent Results Access

## Status

Draft

## Story

**As an** operator,
**I want** the scoring board's controls within thumb reach at the bottom and a results button always available at the top,
**so that** I can operate one-handed and check standings at any time.

## Acceptance Criteria

1. **Given** the scoring board, **When** it renders, **Then** the primary action control(s) are anchored to the bottom of the screen within comfortable thumb reach, ≥44px tall.
2. **Given** the top bar (same layer as the "{n} of 18 confirmed" count), **When** the board renders, **Then** a persistent **View Results** control is present at the top-right at all times — not gated on 18/18 confirmed.
3. **Given** the operator taps the top-right **View Results** at any completion level, **When** the results screen renders, **Then** unentered targets and shots are counted as `0` (per Story 7.1) and rankings reflect that.
4. **Given** the old 18/18-only "View Results" banner, **When** the board renders, **Then** it is removed (superseded by the persistent control) OR retained only as a non-blocking completion confirmation — implementation may keep a subtle "all confirmed" indicator but must not be the only path to results.
5. **Given** a screen width of 360px, **When** the board and its bottom controls render, **Then** there is no horizontal scroll and controls remain reachable without obscuring targets.

## Tasks / Subtasks

- [ ] **Task 1 — Persistent top-right View Results** (AC: 2, 3)
  - [ ] In `ScoringBoardPage.vue`, add a **View Results** control to the subtitle/top bar (the layer showing "{confirmedCount} of 18 confirmed"), aligned right, present at all completion levels.
  - [ ] Tap → `router.push('/archery/results')`. The results page already computes totals treating gaps as 0 once Story 7.1 lands (`ranked()` / `useScores` guard `?? 0`).

- [ ] **Task 2 — Remove/repurpose the 18/18 banner** (AC: 4)
  - [ ] Remove the `v-if="confirmedCount === 18"` "All 18 confirmed / View Results" banner, OR keep only a subtle non-blocking "All confirmed ✓" indicator (no longer the sole results path).

- [ ] **Task 3 — Bottom-anchored controls** (AC: 1, 5)
  - [ ] Anchor the board's primary action(s) to the bottom of the viewport within thumb reach. Use a sticky/fixed bottom bar (`position: sticky; bottom: 0` on a footer container, or `q-page-sticky position="bottom"`), ≥44px tall, full-width, with safe-area padding.
  - [ ] Decide which primary action lives at the bottom: at minimum a prominent **View Results / Done** action mirroring the top control for thumb reach; keep targets scrollable above it without being obscured.
  - [ ] Ensure the bottom bar does not cover the last row of targets — add bottom padding to the grid equal to the bar height.

- [ ] **Task 4 — Responsive check** (AC: 5)
  - [ ] Verify at 360px width: 3-col grid intact, no horizontal scroll, bottom bar reachable, top control not clipped.

- [ ] **Task 5 — Tests** (AC: 1–5)
  - [ ] `ScoringBoardPage.spec.ts`: View Results control present at 0/18 and routes to results; old 18/18-gated banner no longer the only results path; bottom control rendered.

## Dev Notes

### Previous Story Insights
- **Depends on Story 7.1** for results-with-zeros (so View Results is meaningful before 18/18). Pairs with Stories 7.2–7.4 (drawer/editing) but is layout-focused.
- Current `ScoringBoardPage.vue` shows results ONLY via a banner gated on `confirmedCount === 18` (`data-testid="view-results-btn"`). This story makes results reachable at any time from a top-right control and adds bottom-anchored controls for ergonomics.
- The subtitle bar currently renders `{{ confirmedCount }} of 18 confirmed` (`scoring-board-page__subtitle`). The persistent View Results control should live in/next to this layer (FR-7.6: "same layer as the count").

### Source references
- `frontend/src/apps/archery/pages/ScoringBoardPage.vue` — subtitle bar, 18/18 banner, target grid, archer chips, `router.push('/archery/results')`.
- `frontend/src/apps/archery/pages/ResultsPage.vue` — consumes `store.session`; `ranked()` from `useScores.ts` already guards missing scores with `?? 0` (works once partial sessions exist).
- `frontend/src/apps/archery/composables/useScores.ts` — `totalFor`/`ranked` treat missing shots as 0.
- UX mockups: `docs/ux/mockups/scoring-board.html` for visual reference (control placement, theme tokens).

### Ergonomics intent (FR-7.5)
Primary actions belong at the bottom for one-handed outdoor use. The top-right View Results (FR-7.6) is the always-available "check standings" affordance in the status layer. It's acceptable for both a bottom and a top entry to results to coexist; the key constraints are: (a) results reachable at any completion level, (b) primary controls thumb-reachable, (c) targets never obscured.

### Constraints / anti-patterns
- ❌ Don't gate results on 18/18 anymore.
- ❌ Don't let the bottom bar cover targets — pad the grid.
- ❌ Keep ≥44px tap targets and no horizontal scroll at 360px (NFR-1, NFR-2).
- ❌ All navigation via Vue Router; all HTTP via `useApi.ts`.

### Testing
- **Frontend:** Vitest + `@vue/test-utils`. Assert the persistent control exists regardless of `confirmedCount` and routes to `/archery/results`. Manual: verify thumb reach + no target occlusion at 360px and on desktop (6-col grid).
