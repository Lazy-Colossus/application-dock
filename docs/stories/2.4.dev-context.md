### Key decisions
1. Panel state (entries, archerIndex, shotIndex) lives in the component — NOT in the store. Panel is ephemeral; only committed data enters the store.
2. `upsertTarget` is a pure helper inside the store module (not exported). `saveTarget` is the exported action.
3. Initialization: watch `store.scoreEntryOpen` with `{ immediate: true }` — fires on mount if already true.
4. Backdrop dismiss: `q-bottom-sheet :no-backdrop-dismiss="hasAnyEntry"`.
5. Close-button discard guard: q-dialog (already stubbed). Show only if `hasAnyEntry`.
6. Shot-buttons order: `[0, 5, 8]` row 1, `[10, 11]` row 2 (5-cell 3-col grid; 6th cell empty via CSS).
7. reduce-motion: guarded by `typeof window !== 'undefined'` for SSR safety.
8. `saveTarget` is local-only in this story. Story 3.1 fills the HTTP call. Mark with TODO.
9. `shotIndex` typed as plain `number` (0 or 1) — no narrowed union needed in implementation.
10. Back button text: "← Back to {previousName}" where previousName = `roster[archerIndex - 1]` when shotIndex === 0, or `roster[archerIndex]` when shotIndex === 1.

### Risk register
- `entries` ref uses `Record<string, [number | null, number | null]>`. Vue reactivity tracks object keys — use `entries.value = { ...entries.value, [archer]: newPair }` (replace, don't mutate).
- `window.matchMedia` may be undefined in happy-dom. Guard with `typeof window !== 'undefined' && window.matchMedia?.('...')?.matches`.
- `q-dialog` stub in `test/setup.ts` renders slot always (no v-model support). Tests must account for discard dialog content being in the DOM; use `data-testid` to disambiguate.
