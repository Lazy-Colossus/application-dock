# Story 4.1: Session History List

## Status

Ready for Review

## Story

**As an** operator,
**I want** to see a list of all past sessions on the Archery home screen,
**so that** I can quickly find and revisit any previous round.

## Acceptance Criteria

1. **Given** the operator is on the Archery home screen, **When** they tap the History entry point, **Then** they are taken to `HistoryPage.vue` showing all finalised sessions, newest first.
2. **Given** the history list renders, **When** sessions exist, **Then** each entry shows: session label, archer count, winner name, and winning score — all in a single `HistoryListItem` card ≥44px tall.
3. **Given** the `GET /api/archery/sessions` endpoint, **When** called, **Then** it returns a direct array of session summaries (label, archer count, winner name, winning score), sorted newest first; winner and winning score are computed by `archery_service.py`, not the frontend.
4. **Given** `useArcheryHistoryStore`, **When** `loadHistory()` is called, **Then** it sets `loading = true` before the request and `loading = false` in the `finally` block; errors land in `store.error`.
5. **Given** no finalised sessions exist, **When** the history page renders, **Then** an empty state message is shown (e.g. "No sessions yet — finish a round to see it here").
6. **Given** a screen width of 360px, **When** the history list renders, **Then** all entries are readable and tappable without horizontal scrolling.

## Tasks / Subtasks

- [ ] **Task 1 — Backend service: `list_history()`** (AC: 3)
  - [ ] In `backend/app/services/archery_service.py`, add:
    ```python
    def list_history() -> list[SessionSummary]:
        sessions = session_repo.list_sessions()  # already newest-first, excludes _in_progress
        return [_summarise(s) for s in sessions]

    def _summarise(s: SessionData) -> SessionSummary:
        # Tie-break: alphabetical archer name (case-insensitive) when totals match.
        # Sort by (-total, name.casefold()) so highest total comes first; alphabetical breaks ties.
        totals = [(a, _archer_total(s, a)) for a in s.archers]
        totals.sort(key=lambda x: (-x[1], x[0].casefold()))
        winner_name, winner_score = totals[0] if totals else ("", 0)
        return SessionSummary(
            label=s.label,
            archer_count=len(s.archers),
            winner=winner_name,
            winning_score=winner_score,
        )

    def _archer_total(s: SessionData, archer: str) -> int:
        return sum(t.scores.get(archer, [0, 0])[0] + t.scores.get(archer, [0, 0])[1] for t in s.targets)
    ```
  - [ ] Tie-breaking on equal top scores: **alphabetical by archer name, case-insensitive** (uses `str.casefold()` to treat "alice" and "Alice" identically). Document this choice in code.

### API contract — `GET /api/archery/sessions`
[Source: architecture.md#API & Communication Patterns]

- **Response (200):** Direct JSON array — `[ { "label": "...", "archer_count": 3, "winner": "Jamie", "winning_score": 284 }, ... ]`
- **Empty:** `[]` (NOT 404)
- **Sort order:** newest-first by `label` (lexicographic on `YYYY-MM-DD[-N]` works because the date prefix sorts naturally)
- **Field names:** snake_case (`archer_count`, `winning_score`) [Source: architecture.md#Naming Patterns]
- **No envelope.** [Source: architecture.md#Format Patterns]
- **Winner & winning_score computed on the backend** by `archery_service.list_history()` — the FE never recomputes for the list view [AC 3, Source: epics.md FR Coverage Map — Story 4.1]

### Tie-breaking
[Not explicitly in PRD/architecture; PO decision logged 2026-05-28]

If multiple archers share the top total, the winner is the **alphabetically first archer name (case-insensitive)**. Implemented via `sorted(..., key=lambda x: (-total, name.casefold()))`. Document this in `_summarise()` so QA knows what to verify. Rationale: deterministic, fair, and visible to the operator (the roster order is a UI artifact; alphabetical is order-independent).
