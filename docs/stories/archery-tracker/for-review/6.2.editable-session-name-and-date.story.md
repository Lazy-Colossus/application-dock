# Story 6.2: Editable Session Name & Date at Creation

## Status

Ready for Review

## Story

**As an** operator,
**I want** to rename a session when I create it (defaulting to the date),
**so that** I can give memorable names to rounds while still ordering them chronologically.

## Acceptance Criteria

1. **Given** the session setup screen, **When** it renders, **Then** a "Session name" input is shown, preloaded with the date (e.g. `2026-05-29`).
2. **Given** the operator edits the name, **When** they confirm the roster, **Then** the entered name is sent to `POST /api/archery/sessions` and stored as the session `name`; if left blank it falls back to the date default.
3. **Given** a session is created, **When** it is persisted, **Then** its `date` field is set to the current UTC date regardless of the chosen name.
4. **Given** the name is displayed in the UI (resume picker, results, history), **When** rendered, **Then** the custom `name` is shown (not the canonical `label`); the auto-suffix display rule (Story 5.1) applies only when the name is still the date-derived default.
5. **Given** the `CreateSessionRequest` schema, **When** it is validated, **Then** it accepts an optional `name` field alongside `archers`.

## Dev Notes

### Previous Story Insights
- **Depends on Story 6.1** for `name`/`date` on `SessionData` and `CreateSessionRequest`. Do not start until 6.1's schema is merged.
- Story 5.1 established `formatSessionLabel(label)` in `frontend/src/apps/archery/composables/useSessionLabel.ts` — the seam for label display. This story adds a sibling `displaySessionName(session)` so name-vs-label logic lives in one place.
- `useArcherySessionStore` already has `draftRoster`, `draftName` (archer name), `loading`, `error`, `resetDraft()`. Note: `draftName` is the per-archer input — the session name is a **separate** ref to avoid collision.

### Source references
- `frontend/src/apps/archery/pages/SessionSetupPage.vue` — roster entry; `confirmRoster()` calls `store.createSession()`.
- `frontend/src/apps/archery/stores/useArcherySessionStore.ts` — `createSession()` currently posts `{ archers: draftRoster.value }`.
- `frontend/src/apps/archery/composables/useSessionLabel.ts` — display formatting seam.
- `frontend/src/apps/archery/types.ts` — `SessionData`, `SessionSummary` TS interfaces.
- Backend (Story 6.1): `schemas/session.py`, `services/archery_service.py`, `routers/archery.py`.

### Storage vs display
- Canonical `label` (`YYYY-MM-DD[-N]`) is unchanged and still used for filenames, path params, and sort fallback.
- `name` is free text shown to users. `date` (`YYYY-MM-DD`) is the ordering key for history/resume (Stories 6.4, 9.1).

### Constraints / anti-patterns
- ❌ Never use `name` for filenames, routing, or uniqueness — only `label`.
- ❌ Don't reuse the archer-name `draftName` ref for the session name.
- ❌ Don't scatter name-vs-label conditionals across components — use `displaySessionName`.

### Testing
- **Frontend:** Vitest + `@vue/test-utils` + `@pinia/testing`. Co-located `*.spec.ts`.
- **Backend:** pytest — covered primarily under Story 6.1; add a focused test that `POST /sessions` with `name` persists it and `date` is set to today.
- **Manual:** create a session named "Club Champs", finalise, confirm history shows "Club Champs" but the file on disk is still `YYYY-MM-DD[-N].json`.
