## Decisions

1. **Pydantic v2 syntax throughout.** Use `field_validator` (mode='after' for cross-value access via `info.data`) and `model_validator` for invariants spanning multiple fields. Avoid the deprecated v1 `@validator`.

2. **Shot value enum vs. plain `Literal[0, 5, 8, 10, 11]`.** Going with a `Literal` typing on the validator side, not a `IntEnum`. Reason: the wire format is a plain integer per architecture.md#Format Patterns. A `Literal` validator gives the same type-safety without changing serialization.

3. **`SessionStatus` as `Literal["in_progress", "finalised"]`.** Single source of truth at the type level; no enum class needed.

4. **`TargetScores.scores` validation.** The shape is `dict[str, list[int]]`. Validate:
   - Every value list has exactly 2 ints.
   - Every int is in `{0, 5, 8, 10, 11}`.
   - Validator on the `SessionData` level (not `TargetScores`) checks the cross-cutting invariant that `scores.keys() == set(archers)` — i.e. every archer in the roster has scores for this target, and no stray archer names appear. This is mandatory because Story 2.4's confirm-target flow REQUIRES all archers to have shots before persisting; the schema enforces the same.

5. **Archer-name validation on `SessionData`.** Length ≥ 1; each entry non-empty after strip; unique within the list (case-sensitive per FR-2.2 — "Alice" and "alice" are different archers). I'll trim whitespace as a normalization step in the validator, not just reject — but the trimmed value is what gets stored. Document in code comment.

6. **`SessionData.created`.** Stored as ISO 8601 string per architecture.md#Format Patterns. I'll type it as `str` (not `datetime`) so JSON round-trips are lossless and there's no timezone-conversion drift. Validator regex-checks the format. Format: `YYYY-MM-DDTHH:MM:SSZ` (UTC, trailing Z) — same shape used in the architecture's example JSON.

7. **`SessionSummary` lives here, not Story 4.1.** Story spec puts it in this story explicitly. Pydantic model with `label`, `archer_count`, `winner`, `winning_score`. Computation of these is in `archery_service.py` (Story 4.1) — schema is field-shape only.

8. **`CreateSessionRequest` lives here.** Story 2.2 will import it from `app.schemas.session`. Defining it now keeps the schema file complete.

9. **`session_repo.py` is path-agnostic for `write_session`.** Caller supplies the path. This lets Stories 2.5 (finalise to `YYYY-MM-DD[-N].json`) and 3.1 (`_in_progress.json`) both reuse the same writer. `read_session(label)` resolves `data_dir / f"{label}.json"` itself.

10. **Skip files starting with `_` in `list_sessions()`.** Excludes `_in_progress.json` and any other dot-prefixed temp/system files. Mirrors the architecture's "underscore = system file" convention.

11. **No HTTP exceptions in the repo.** Raise `FileNotFoundError` / `ValueError`; routers translate. Confirmed by architecture.md#Service Boundaries.

12. **Atomic write helper is module-private.** `_atomic_write_json(path, payload)` — leading underscore signals private. Both `write_session` and `write_in_progress` call it.

13. **`list_sessions` returns `list[SessionData]`, not summaries.** The story spec calls it `list_sessions()` returning all finalised sessions. The summary computation belongs in `archery_service.py` (Story 4.1). Repo is dumb file I/O.

## Risk register

| Risk | Mitigation |
|------|------------|
| Pydantic v2 validator subtleties (mode='before' vs 'after', `info.data`) | Use mode='after' by default; only use 'before' when transforming input strings (e.g. archer-name strip). |
| `dict[str, list[int]]` JSON serialization preserving insertion order | Python 3.7+ dicts preserve order; `model_dump(mode='json')` honors this. ✓ |
| `os.replace()` on macOS test environments | Standard POSIX semantics — atomic. Tests run on darwin so coverage is real. |
| `Path.glob("*.json")` returns iterator with non-guaranteed order | Always sort explicitly: `sorted(..., key=lambda p: p.name, reverse=True)`. |
| `_in_progress.json` accidentally returned by `list_sessions()` if the underscore filter regresses | Test explicitly: seed `_in_progress.json` AND `_other.json`; assert neither appears in list. |
| Pre-existing temp dir from `conftest.py` polluted across tests | Each test that touches the filesystem uses `monkeypatch.setattr(settings, "data_dir", tmp_path)` to scope to its own `tmp_path`. |
