### Key decisions
1. `TargetIcon` is a native `<button>` (not `q-btn`) — needed for `aria-label` and clean disabled-state semantics.
2. Unconfirmed style: inline via `:style` binding (CSS vars may not be guaranteed in happy-dom test env; use explicit hex values).
3. Confirmed style: `background: #2E7D32`, number color `#A5D6A7`.
4. 3-col grid mobile → 6-col desktop via CSS media query.
5. The `ScoreEntryPanel` stub uses `q-bottom-sheet v-model="store.scoreEntryOpen"` — already stubbed in `test/setup.ts`, no new stubs needed.
6. `ScoringBoardPage` mount guard: if `store.session === null` → `router.replace('/archery')`. Use `replace` not `push` (no back-nav to a broken board).
7. Subtitle font-size: 13px (match mockup HTML, overrides DESIGN.md 12px).

### Risk register
- `TargetIcon` must be a `<button>` not a Quasar component — the stub in tests needs to wrap accordingly. Use local stub in spec file.
- `ScoringBoardPage` null-guard must NOT run if navigating away (router.replace inside onMounted is fine).
- Store test: `targetByNumber` and `isConfirmed` need `session.value` to be non-null — initialize in test fixture.
