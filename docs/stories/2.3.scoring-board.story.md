# Story 2.3: Scoring Board

## Status

Ready for Review

## Story

**As an** operator,
**I want** a visual board of 18 numbered target icons I can tap in any order,
**so that** I can score targets as my group reaches them on the field.

## Acceptance Criteria

1. **Given** the operator is on the scoring board, **When** it renders, **Then** 18 target icons numbered 1тАУ18 are displayed; each is at minimum 44px in both dimensions.
2. **Given** a target has not yet been confirmed, **When** it is displayed, **Then** the icon appears in its default (unconfirmed) state.
3. **Given** a target has been confirmed, **When** it is displayed, **Then** the icon appears green.
4. **Given** the operator taps any unconfirmed target, **When** the tap registers, **Then** the score-entry panel opens for that target.
5. **Given** the operator taps a confirmed (green) target, **When** the tap registers, **Then** the target re-opens in the score-entry panel with its previously entered scores pre-populated; the green state resets until re-confirmed.
6. **Given** a screen width of 360px, **When** the scoring board renders, **Then** all 18 icons are visible and tappable without horizontal scrolling.

### Behavioral spec
[Source: EXPERIENCE.md#Component Patterns, #State Patterns]

- **Tap unconfirmed target:** opens Score Entry Panel for that target
- **Tap confirmed (green) target:** opens Score Entry Panel with previously entered scores pre-populated; the green state resets until the operator re-confirms (FR-2.12; visual reset is handled by the panel's confirm flow in Story 2.4)
- **Targets can be entered in any order** (FR-2.6) тАФ no enforced sequence
- **All 18 confirmed тЖТ "View Results" banner:** NOT in this story тАФ implemented in Story 2.5 along with the results screen
- **No long-press, no swipe** тАФ tap only [Source: EXPERIENCE.md#Interaction Primitives]
- **Reduce motion:** if user has `prefers-reduced-motion`, the panel should appear without slide-up animation (Story 2.4's responsibility тАФ flag it here but implement in 2.4)

### Accessibility floor
[Source: EXPERIENCE.md#Accessibility Floor]

- Focus traversal: 1 тЖТ 2 тЖТ 3 тЖТ ... тЖТ 18 in reading order

- Tap targets тЙе44px (56px in this design)
