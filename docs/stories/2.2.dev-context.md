### Key decisions locked

1. `created` timestamp uses `datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%SZ")` — schema regex requires trailing `Z`, not `+00:00`.
2. Label date is UTC (`date.today()` is called without tz-awareness but interpreted as server-clock UTC since the backend runs in UTC).
3. `generate_session_label` scans for `YYYY-MM-DD.json`, `YYYY-MM-DD-2.json`, … in DATA_DIR AND checks `_in_progress.json`'s stored label for collision.
4. Service raises `RuntimeError("session already in progress")` on conflict; router converts to HTTP 409.
5. No display formatting (`-2` → `#2`) in this story — deferred to 5.1.
6. Duplicate checking in the frontend roster is **case-sensitive** (exact string match, per FR-2.2).

### Risk register
- `generate_session_label` must handle gap in suffix sequence (e.g., `-2` exists but `-3` does not → return `-3`). No, it should just scan from 2 upward and return the first missing one. No gaps expected; files are written sequentially.
- `_in_progress.json` collision check: must read the file's `label` field, not just check file existence.
- `q-input` is used in `SessionSetupPage.vue` — add it to `test/setup.ts` stubs or tests will crash.
