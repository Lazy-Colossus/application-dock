# Story 9.2: Thematic Monochrome App Icon

## Status

Draft

## Story

**As an** operator,
**I want** the Archery app's icon to be a thematic target/bow glyph in a single colour,
**so that** the app reads clearly and on-theme on the shell landing page.

## Acceptance Criteria

1. **Given** `src/apps/registry.ts`, **When** the archery entry is defined, **Then** its `icon` is a thematic Material Icons glyph (e.g. `my_location` / `gps_fixed` target reticle, or `adjust`) rather than `sports_score`.
2. **Given** the shell landing page, **When** the Archery `AppCard` renders, **Then** the icon is displayed monochrome (single theme colour, no multi-tone).
3. **Given** the chosen icon, **When** reviewed, **Then** it is recognisably archery/target-themed and legible at the card's icon size.
4. **Given** the change is icon-only, **When** implemented, **Then** no other registry fields or routes change.

## Tasks / Subtasks

- [ ] **Task 1 — Swap the registry icon** (AC: 1, 4)
  - [ ] In `frontend/src/apps/registry.ts`, change the archery entry's `icon` from `'sports_score'` to a thematic target glyph. Recommended: `'adjust'` (concentric-circle target) or `'gps_fixed'`/`'my_location'` (reticle). All are single-tone Material Icons.
  - [ ] Leave `id`, `label`, `route` unchanged.

- [ ] **Task 2 — Confirm monochrome rendering** (AC: 2, 3)
  - [ ] Check `AppCard.vue` renders the icon via `q-icon` (inherits a single theme colour). If any multi-tone/filled variant is used, ensure a single-colour token (e.g. accent or ink) applies.
  - [ ] Verify legibility at the card's icon size against the Carbon dark theme.

- [ ] **Task 3 — Tests / snapshot** (AC: 1)
  - [ ] If `AppCard.spec.ts` or a registry test asserts the icon name, update it to the new glyph. Otherwise add a tiny assertion that the archery descriptor's `icon` is the new value.

## Dev Notes

### Previous Story Insights
- Story 1.2 built the shell: `registry.ts` (`AppDescriptor[]`) is the single source of truth, and `HomePage.vue` renders one `AppCard.vue` per entry using the `icon` field. The archery entry currently uses `'sports_score'`.
- This is the smallest possible change — one string in the registry — plus a confirmation that rendering is monochrome.

### Source references
- `frontend/src/apps/registry.ts` — archery `AppDescriptor` (`icon: 'sports_score'`).
- `frontend/src/components/AppCard.vue` — renders the icon (Quasar `q-icon`, Material Icons set per `tech-stack.md`).
- `frontend/src/pages/HomePage.vue` — iterates registry → `AppCard`.
- `tech-stack.md` — "Icons: Material Icons (Quasar default set)". Pick from that set so no asset import is needed.

### Icon choice
The request is "a target or a bow or something more thematic, but still monocolour." Material Icons options (all single-tone): `adjust` (target rings — strong match), `gps_fixed`, `my_location`, `track_changes`. There is no dedicated bow glyph in the default set, so a target reticle is the closest thematic, legible choice. Dev may pick among these; `adjust` is the recommended default.

### Constraints / anti-patterns
- ❌ Don't introduce a custom SVG/asset pipeline for v1 — use the Material Icons set already in use.
- ❌ Don't change `label`/`route`/`id` or any routing.
- ❌ Keep it monochrome — no multi-tone/duotone icon variant.

### Testing
- **Frontend:** Vitest. Trivial assertion on the registry value and/or `AppCard` rendering. Manual: load the shell landing page and confirm the on-theme monochrome target icon.
